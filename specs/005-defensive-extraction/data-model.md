# Phase 1 Data Model: Defensive Structured Extraction with JSON Schemas

All entities below are implemented as pydantic v2 models at
`katas/005_defensive_extraction/models.py`. Validation runs on construction;
any invalid payload raises `pydantic.ValidationError` and is surfaced to the
caller with the offending field path (FR-008). The JSON Schema generated by
`ExtractedRecord.model_json_schema()` is the same schema passed to the
extraction tool as `input_schema` (FR-001) — a single source of truth (D-002).

## ExtractedRecord

The record returned by a successful extraction. Conforms exactly to its
declared schema. Required fields are present and non-null; optional fields
are either typed values or `None`; enumerated fields contain a declared
option or the escape option. `extra="forbid"` (FR-010).

```python
class ExtractedRecord(BaseModel):
    model_config = ConfigDict(extra="forbid")

    # Required fields — guaranteed present in every source (FR-003)
    subject: str                            # The entity the record describes
    reported_at: str                        # ISO-8601 date/time as observed in source

    # Optional fields — nullable unions per FR-004 (NOT bare `str`)
    nickname: Optional[str] = None          # Defended by schema lint (D-003)
    location: Optional[str] = None

    # Enumerated field with escape option per FR-005
    status: Literal["active", "inactive", "other", "unclear"]
    # Paired details field — populated whenever status == "other" or "unclear"
    status_details: Optional[str] = None    # AmbiguityMarker pair (FR-005)
```

| Field | Required? | Type | FR / SC |
|-------|-----------|------|---------|
| `subject` | yes | `str` | FR-003 |
| `reported_at` | yes | `str` (ISO-8601) | FR-003 |
| `nickname` | no | `Optional[str] = None` | FR-004, SC-004 |
| `location` | no | `Optional[str] = None` | FR-004, SC-004 |
| `status` | yes | `Literal["active","inactive","other","unclear"]` | FR-005, SC-003 |
| `status_details` | conditional | `Optional[str] = None` | FR-005, SC-003 |

**Invariants (enforced by `model_validator(mode="after")`)**
1. If `status in {"other", "unclear"}`, `status_details` MUST be a non-empty
   string. Enforces the AmbiguityMarker pairing declared in the spec
   (FR-005).
2. `extra="forbid"` rejects any key the model did not declare (FR-010).
3. On `ValidationError`, the raised exception's `errors()` output is the
   payload surfaced to the caller (FR-008).

**Anti-pattern defense (FR-004)**
- The schema-lint test
  (`tests/katas/005_defensive_extraction/unit/test_schema_lint.py`)
  walks `ExtractedRecord.model_fields.items()`. For any field whose
  `FieldInfo.is_required()` returns `False`, the test asserts the annotated
  type's union includes `type(None)` AND the non-None arm is not a bare `str`
  without a `Literal` or other narrower type. Failure message points at FR-004
  and names the offending field.

## SchemaDefinition

A frozen snapshot of the extraction tool's schema, ready to hand to
`messages.create(...)`.

| Field | Type | Notes |
|-------|------|-------|
| `name` | `Literal["extract_record"]` | Fixed for this kata. |
| `description` | `str` | Human-facing tool description (FR-009, Principle VIII). |
| `input_schema` | `dict[str, Any]` | Frozen output of `ExtractedRecord.model_json_schema()`. |
| `required_fields` | `tuple[str, ...]` | Echo of the `required` array, for audit. |
| `optional_fields` | `tuple[str, ...]` | Complement of `required_fields`; audited by the schema lint. |
| `escape_options` | `dict[str, tuple[str, ...]]` | Per enumerated field, the subset of enum values that are escape values (FR-005). |

**Invariants**
1. Every field in `required_fields` MUST NOT be nullable in
   `input_schema` (`"null"` not in the field's JSON Schema `type`). FR-003.
2. Every field in `optional_fields` MUST be nullable in `input_schema`
   (`"null"` in the field's JSON Schema `type`). FR-004.
3. Every enumerated field MUST have at least one entry in `escape_options`
   (FR-005).

## AmbiguityMarker

The combination of an enumerated field set to its escape option and its
paired details field populated. A first-class, auditable signal of model-
side ambiguity — never a hidden coercion (spec Key Entities).

| Field | Type | Notes |
|-------|------|-------|
| `field_name` | `str` | e.g. `"status"`. |
| `escape_value` | `Literal["other", "unclear"]` | Declared escape option. |
| `observed_value` | `str` | The raw phrase from the source that triggered the escape. |
| `details` | `str` (non-empty) | Human-readable capture for downstream review. |

Constructed by `AmbiguityMarker.from_record(record: ExtractedRecord, field_name: str)`
whenever the corresponding field is set to an escape value (FR-005).

## FabricationMetric

A per-run counter operationalizing SC-001. Computed by walking a single
extracted record against a labeled fixture's `null_map`.

| Field | Type | Notes |
|-------|------|-------|
| `fixture_id` | `str` | Fixture case directory name (e.g., `missing_optional`). |
| `labeled_null_fields` | `tuple[str, ...]` | Optional fields the labeler marked absent from the source. |
| `observed_non_null_fields` | `tuple[str, ...]` | Subset of `labeled_null_fields` that came back non-null (fabrications). |
| `fabrication_count` | `int` | `len(observed_non_null_fields)`. |
| `null_rate` | `float` | `1 - fabrication_count / max(len(labeled_null_fields), 1)`. SC-004. |

**Computation** — `FabricationMetric.compute(record, fixture_labels)`:

```text
for field in fixture_labels.null_map:
    if getattr(record, field) is not None:
        observed_non_null_fields.append(field)
fabrication_count = len(observed_non_null_fields)
```

**Assertions driven by this metric**
- SC-001: across the full labeled corpus, `sum(m.fabrication_count) == 0`.
- SC-004: across the full labeled corpus, `mean(m.null_rate) >= 0.99`.
- SC-003: a parallel `escape_map` audit asserts 100% of ambiguous fixtures
  routed through the escape enum (driven by `AmbiguityMarker` presence).

## Relationships

```
SchemaDefinition
  └── input_schema  ← ExtractedRecord.model_json_schema()

Extraction run (one fixture)
  ├── ExtractedRecord                  (the returned record; validated)
  ├── AmbiguityMarker[]                (zero or more, one per escape-routed field)
  └── FabricationMetric                (computed against fixture_labels)
```

## What is deliberately NOT modeled

- Retry / re-extraction logic on validation failure — out of scope per
  FR-006 (validation failures are surfaced, not auto-corrected).
- Multi-record extraction (list of records per source) — scope is one
  record per source for this kata.
- Provenance span pointers (character offsets back to the source) — deferred;
  FR-008 requires enough context to audit, not span-level provenance.
  Revisit at a later kata if a spec demands it.
